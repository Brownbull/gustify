# Story 1.4: Recipe Search & Filtering

## Intent
**Epic Handle:** "Your pantry-ranked recipe library"
**Story Handle:** "This story adds tabs and a search bar to the recipe library so you can find exactly what you want."

## Status: drafted

## User Story
As a user, I want to search recipes by name, cuisine, or ingredient, and filter by complexity level, so that I can find specific recipes quickly.

## Acceptance Criteria

### AC-1: Text Search
- **Given** the recipe list
- **When** I type "pollo" in the search bar
- **Then** only recipes containing "pollo" in name, description, or ingredient names are shown

### AC-2: Cuisine Filter
- **Given** the recipe list
- **When** I select cuisine filter "Chilena"
- **Then** only recipes with cuisine "Chilena" are shown

### AC-3: Complexity Filter
- **Given** the recipe list
- **When** I select complexity "1-2" (Facil)
- **Then** only recipes with complexity 1 or 2 are shown

### AC-4: Combined Filters
- **Given** active search text + cuisine + complexity filters
- **When** all are applied simultaneously
- **Then** results match ALL criteria (AND logic)

### AC-5: Filter Reset
- **Given** active filters producing a subset of recipes
- **When** I clear all filters
- **Then** the full recipe list is shown again

### AC-6: Zero Results
- **Given** filters that match no recipes
- **When** applied
- **Then** a "No se encontraron recetas" message is shown with a clear filters button

## Architectural ACs

- **AC-ARCH-LOC-1:** Filter component at `src/components/RecipeFilters.tsx`
- **AC-ARCH-PATTERN-1:** Filtering is client-side (all recipes already loaded via subscription from Story 1.3)
- **AC-ARCH-PATTERN-2:** Search input sanitized before use (truncate to 100 chars, strip special regex chars)
- **AC-ARCH-NO-1:** No new Firestore queries for filtering — client-side only on the subscribed set

## Dev Notes

- All filtering is client-side. The recipe pool is loaded via real-time subscription (Story 1.3). No additional Firestore queries needed.
- Cuisine list is derived dynamically from the loaded recipes (`[...new Set(recipes.map(r => r.cuisine))]`). No hardcoded cuisine list.
- The search is case-insensitive and matches against recipe name, description, and ingredient names.
- Consider using `useMemo` for the filtered list computation to avoid unnecessary recalculations.
- Spanish labels: "Buscar recetas..." (placeholder), "Todas" (all), "Facil" (easy), "Media" (medium), "Dificil" (hard), "Limpiar filtros" (clear filters), "No se encontraron recetas" (no recipes found).
