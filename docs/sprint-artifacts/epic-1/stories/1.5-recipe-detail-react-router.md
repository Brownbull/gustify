# Story 1.5: Recipe Detail View & React Router

## Intent
**Epic Handle:** "Your pantry-ranked recipe library"
**Story Handle:** "This story puts each recipe on its own page and gives the whole app proper street addresses."

## Status: drafted

## User Story
As a user, I want to view a recipe's full details in a dedicated view with proper URL routing, so that I can follow cooking instructions and navigate between views.

## Acceptance Criteria

### AC-1: Recipe Detail Navigation
- **Given** a recipe card in the list
- **When** I tap it
- **Then** I navigate to `/recipes/:id` showing full recipe details

### AC-2: Ingredient Availability Display
- **Given** the recipe detail view
- **When** viewing the ingredient list
- **Then** each ingredient shows quantity, unit, name, and a green (have) or red (missing) pantry indicator

### AC-3: Sequential Steps
- **Given** the recipe detail view
- **When** viewing cooking steps
- **Then** steps are displayed in order with step numbers and clear instructions

### AC-4: Recipe Metadata
- **Given** the recipe detail view
- **When** viewing the header
- **Then** it shows name, description, cuisine, complexity (1-5), prep time, cook time, servings, and techniques

### AC-5: Bottom Navigation via Routes
- **Given** any view in the app
- **When** I tap a bottom nav tab
- **Then** I navigate to the correct URL route (/, /pantry, /recipes, /map-items)

### AC-6: Direct URL Navigation
- **Given** a direct URL like `/recipes/abc123`
- **When** I navigate to it
- **Then** the recipe detail loads (or shows "not found" if invalid)

### AC-7: Recipe Not Found
- **Given** an invalid recipe ID in the URL
- **When** I navigate to `/recipes/invalid-id`
- **Then** a "Receta no encontrada" message is shown with a link back to recipes

## Architectural ACs

- **AC-ARCH-LOC-1:** RecipeDetailPage at `src/pages/RecipeDetailPage.tsx`
- **AC-ARCH-PATTERN-1:** React Router v6 with `createBrowserRouter` or `<BrowserRouter>` + `<Routes>`
- **AC-ARCH-PATTERN-2:** Bottom nav uses `<NavLink>` for active state styling
- **AC-ARCH-NO-1:** No state-based view switching in App.tsx — all routing via React Router
- **AC-ARCH-PATTERN-3:** Recipe loaded by ID from recipeStore (already subscribed) or fallback to `getRecipeById()`
- **AC-ARCH-LOC-2:** `data-testid` on all interactive elements and key display areas

## Dev Notes

- This is the largest story in Epic 1. The React Router migration touches every page because it replaces the state-based `AppView` routing in App.tsx.
- Remove the `onNavigateToMap` and `onNavigateToPantry` callback props from pages — they're no longer needed with React Router's `useNavigate()`.
- The `UserHeader` component in App.tsx can stay as a layout element rendered above `<Outlet>` or `<Routes>`.
- Keep the pantry subscription logic in App.tsx (or move to a layout component) — it needs to stay active across all routes.
- The existing `RecipeDetailModal` in RecipesPage.tsx is deleted — replaced by the full RecipeDetailPage.
- Firebase Hosting needs `rewrites` configuration for SPA routing (all routes -> index.html). Check `firebase.json` for existing rewrite config.
- The "Cooked it!" button is NOT added here — that's Epic 4, Story 4.1.
