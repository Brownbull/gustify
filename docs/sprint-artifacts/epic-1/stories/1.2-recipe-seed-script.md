# Story 1.2: Recipe Seed Script & Initial Pool

## Intent
**Epic Handle:** "Your pantry-ranked recipe library"
**Story Handle:** "This story stocks the recipe library with its first 50 books."

## Status: review

## User Story
As a user, I want a pre-populated recipe pool with Chilean home cooking recipes, so that I have something to browse from day one.

## Acceptance Criteria

### AC-1: Seed Script Populates Recipes
- **Given** the seed script is executed against staging
- **When** it completes successfully
- **Then** 50+ recipes exist in the `recipes/` Firestore collection

### AC-2: Recipes Reference Canonical Ingredients
- **Given** a seeded recipe with ingredients
- **When** an ingredient matches a canonical ingredient
- **Then** it includes the `canonicalId` referencing the canonical ingredients dictionary

### AC-3: Idempotent Execution
- **Given** the seed script has already been run
- **When** executed again
- **Then** no duplicate recipes are created (checks by normalized name before writing)

### AC-4: Firestore Rules Allow Reads
- **Given** an authenticated user
- **When** they query the `recipes/` collection
- **Then** read access is granted; direct write access is denied

## Architectural ACs

- **AC-ARCH-LOC-1:** Seed data at `scripts/data/seed-recipes.ts`
- **AC-ARCH-LOC-2:** Seed runner at `scripts/seed-recipes.ts`
- **AC-ARCH-PATTERN-1:** Follows existing `scripts/seed-ingredients.ts` pattern (Admin SDK, idempotent, batch writes)
- **AC-ARCH-PATTERN-2:** Batch writes chunked at 500 operations (Firestore limit)
- **AC-ARCH-LOC-3:** Firestore rules updated in `firestore.staging.rules` and `firestore.rules`

## Dev Notes

- Use the exact same Firebase Admin SDK initialization pattern from `scripts/seed-ingredients.ts`.
- Recipe names should be in Chilean Spanish. Descriptions should be brief (1-2 sentences).
- For canonicalId mapping: import the canonical ingredients array from `scripts/data/canonical-ingredients.ts` and match by Spanish name. Not all recipe ingredients will have a canonical match (e.g., spice blends, sauces) — that's fine, leave `canonicalId` undefined for those.
- The seed data file will be large (~50 recipes). Consider organizing by cuisine/category with section comments.
- This script will be run once per environment. The idempotency check ensures safety on re-runs.
