# Story 4.3: Cooking History View

## Intent
**Epic Handle:** "Cook it, log it, grow from it"
**Story Handle:** "This story adds a photo album to your cooking passport -- flip through what you've made."

## Status: drafted

## User Story
As a user, I want to see my cooking history (list of cooked meals with dates and ratings), so that I can remember what I've made.

## Acceptance Criteria

### AC-1: History List
- **Given** the More page
- **When** I view the cooking history section (or tap "Historial de cocina")
- **Then** I see a chronological list of cooked meals (most recent first)

### AC-2: Meal Entry Display
- **Given** a cooked meal in the history
- **When** displayed
- **Then** it shows: recipe name, date (formatted), rating (stars), cuisine badge, complexity badge

### AC-3: Empty State
- **Given** no cooked meals
- **When** viewing history
- **Then** an empty state shows "Aun no has cocinado nada. Busca una receta y pruebala!"

### AC-4: Pagination
- **Given** more than 20 cooked meals
- **When** scrolling
- **Then** older meals load incrementally (infinite scroll or "Cargar mas" button)

## Architectural ACs

- **AC-ARCH-LOC-1:** CookingHistory component at `src/components/CookingHistory.tsx`
- **AC-ARCH-PATTERN-1:** Data loaded from cookingLogStore (already fetched in Story 4.1)
- **AC-ARCH-PATTERN-2:** Pagination via Firestore `startAfter` cursor on date field

## Dev Notes

- Keep this component simple. It's a read-only list.
- Date formatting in Spanish: use `Intl.DateTimeFormat('es-CL', { dateStyle: 'medium' })` or a relative time formatter.
- The star rating display can be simple text stars or icons. No interactivity needed -- this is display only.
- Pagination cursor uses the `date` field. Firestore `startAfter(lastDate)` with `orderBy('date', 'desc')` and `limit(20)`.
- The `cookedMeals` subcollection is user-scoped (`users/{uid}/cookedMeals`), so queries are naturally filtered to the current user.
