# Story 4.5: Prepared Meal Logging & Navigation Polish

## Intent
**Epic Handle:** "Cook it, log it, grow from it"
**Story Handle:** "This story labels each stamp in your passport and polishes the app's front door."

## Status: drafted

## User Story
As a user, I want to log meals against meal types and navigate a complete 5-tab app, so that my cooking data is organized and the app feels complete.

## Acceptance Criteria

### AC-1: Meal Type Selection
- **Given** the cooking log flow (after "Lo cocine!" and rating)
- **When** I'm confirming the meal
- **Then** I can optionally select a meal type: Desayuno, Almuerzo, Once, Cena

### AC-2: Missing Meal Type Suggestion
- **Given** the meal type selector
- **When** none of the options match
- **Then** I can tap "Otro" and type a suggestion that is saved for future review

### AC-3: Five-Tab Navigation
- **Given** the bottom navigation
- **When** rendered
- **Then** 5 tabs are visible: Inicio, Despensa, Recetas, Mapear, Mas

### AC-4: Active Tab State
- **Given** I'm on a specific page
- **When** viewing the bottom nav
- **Then** the current tab is visually highlighted

### AC-5: Spanish Labels Throughout
- **Given** any page in the app
- **When** viewing user-facing text
- **Then** all labels, buttons, and messages are in Spanish

## Architectural ACs

- **AC-ARCH-PATTERN-1:** Meal type stored as optional field on `CookedMeal` document
- **AC-ARCH-PATTERN-2:** Meal type suggestions stored in `mealTypeSuggestions/{id}` collection
- **AC-ARCH-LOC-1:** BottomNav already exists from Story 1.5 -- verify 5 tabs including "Mas"
