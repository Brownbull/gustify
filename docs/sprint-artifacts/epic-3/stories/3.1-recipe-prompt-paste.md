# Story 3.1: Recipe Prompt Assembly & External Paste Path

## Intent
**Epic Handle:** "Anyone can grow the recipe pool"
**Story Handle:** "This story builds the drop-off window for the community recipe box -- bring your own AI."

## Status: drafted

## User Story
As a user, I want to generate a recipe prompt from my context and paste back a structured result from any AI, so that I can add recipes using my preferred AI tool.

## Acceptance Criteria

### AC-1: Prompt Assembly
- **Given** my pantry items, dietary restrictions, proficiency tier, and optionally a desired cuisine or ingredient
- **When** I tap "Generar prompt"
- **Then** a structured prompt is assembled including my context and the expected JSON output format specification

### AC-2: Copy to Clipboard
- **Given** a generated prompt
- **When** I tap "Copiar"
- **Then** the prompt is copied to the clipboard with a confirmation toast

### AC-3: Paste & Parse
- **Given** a valid JSON recipe response from any AI
- **When** I paste it into the text area and tap "Validar"
- **Then** the JSON is parsed and validated via the RecipeSchema (Zod)

### AC-4: Preview Before Submit
- **Given** a valid parsed recipe
- **When** validation passes
- **Then** a preview shows the recipe (name, cuisine, ingredients, steps) with a "Guardar" button

### AC-5: Parse Error Feedback
- **Given** a malformed or incomplete JSON response
- **When** I paste and validate
- **Then** field-level error messages identify what's missing or invalid (FR-5.12)

### AC-6: No Results Flow
- **Given** a search in RecipesPage that returns no results
- **When** I see the empty state
- **Then** a "Agregar receta" link navigates me to the contribution page (FR-4.9)

## Architectural ACs

- **AC-ARCH-LOC-1:** Prompt builder at `src/lib/prompt-builder.ts` (pure function)
- **AC-ARCH-LOC-2:** Contribute page at `src/pages/ContributeRecipePage.tsx`
- **AC-ARCH-LOC-3:** Paste form at `src/components/RecipePasteForm.tsx`
- **AC-ARCH-LOC-4:** Recipe preview at `src/components/RecipePreview.tsx`
- **AC-ARCH-PATTERN-1:** Prompt builder is a pure function — no side effects, fully testable
- **AC-ARCH-PATTERN-2:** Zod validation produces `ServiceResult<Recipe>` with field-level errors
- **AC-ARCH-NO-1:** Prompt builder does NOT call any API — it only assembles text

## Dev Notes

- The prompt builder is the core innovation of this story. It assembles the user's full context into a prompt that works with ANY AI (ChatGPT, Claude, Gemini, local models). The prompt IS the product.
- The expected output format in the prompt should match `RecipeSchema` exactly. Include a concrete example output in the prompt to improve AI compliance.
- Clipboard API (`navigator.clipboard.writeText()`) requires HTTPS or localhost. Works on both staging and production.
- The "Guardar" button in RecipePreview will call the `contributeRecipe` Cloud Function from Story 3.2. For now, it can show a "coming soon" state or be wired up if Story 3.2 is implemented first.
- The ContributeRecipePage uses tabs but starts with only the "Prompt externo" tab active. Stories 3.2 and 3.3 add the other tabs.
- Consider adding optional fields to the prompt form: "Quiero cocinar..." (desired cuisine) and "Tengo ganas de usar..." (desired ingredient) to guide the AI.
