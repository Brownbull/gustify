# Story 2.2: Dietary Profile Settings UI

## Intent
**Epic Handle:** "No unsafe recipe reaches the screen"
**Story Handle:** "This story installs the bouncer's rulebook -- what to block, what to allow."

## Status: drafted

## User Story
As a user, I want to set my dietary restrictions and food allergies on my profile, so that recipes are filtered for my safety.

## Acceptance Criteria

### AC-1: Set Dietary Restrictions
- **Given** the profile/settings page
- **When** I select dietary restrictions (e.g., "Sin gluten", "Vegetariano", "Vegano", "Bajo en carbohidratos")
- **Then** my selections are saved to my Firestore user profile under `cookingProfile.dietPrefs`

### AC-2: Set Food Allergies
- **Given** the profile/settings page
- **When** I select food allergies (e.g., "Mariscos", "Frutos secos", "Lacteos", "Huevo", "Soja", "Pescado")
- **Then** my selections are saved to `cookingProfile.allergies`

### AC-3: Persistence
- **Given** saved dietary preferences
- **When** I reopen the app or return to settings
- **Then** my previously saved restrictions and allergies are shown as selected

### AC-4: Immediate Updates
- **Given** I change my dietary restrictions
- **When** I toggle a restriction on or off
- **Then** the change is saved to Firestore immediately (no separate "save" button needed)

### AC-5: Navigation
- **Given** the bottom navigation
- **When** a "More" or profile option is accessible
- **Then** I can reach the dietary settings (via MorePage or profile section)

## Architectural ACs

- **AC-ARCH-LOC-1:** MorePage at `src/pages/MorePage.tsx`
- **AC-ARCH-LOC-2:** DietaryProfileForm at `src/components/DietaryProfileForm.tsx`
- **AC-ARCH-PATTERN-1:** Profile updates go through `src/services/auth.ts` (add `updateCookingProfile()`)
- **AC-ARCH-PATTERN-2:** AuthStore reflects updated cookingProfile after save
- **AC-ARCH-NO-1:** No direct Firestore imports in components

## Dev Notes

- The `CookingProfile` type already has `dietPrefs: string[]` and `allergies: string[]` — no type changes needed.
- The `ensureUserProfile()` in auth.ts already creates a default `cookingProfile` with empty arrays on first sign-in.
- Dietary restrictions (`dietPrefs`) are descriptive labels used for prompt generation. Allergies map to the allergen taxonomy from Story 2.1 and are used for strict filtering.
- Auto-save on toggle (no submit button) matches mobile UX conventions. Use debounce if needed to avoid excessive writes.
- The MorePage is a placeholder for now — it will grow to include cooking history, exploration profile, etc. in Epic 4.
- Spanish labels for dietary restrictions: "Sin gluten", "Vegetariano", "Vegano", "Bajo en carbohidratos", "Sin lactosa".
- Spanish labels for allergies: "Mariscos", "Frutos secos", "Lacteos", "Huevo", "Soja", "Pescado", "Gluten".
